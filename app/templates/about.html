What ARIA does

Core tools for operating multiple compatible AI endpoints from one private dashboard.

Provider monitoringTrack endpoint health, response latency, compatibility, and check history.
Model discoveryLoad the current model catalog from OpenAI or Anthropic compatible endpoints.
Inference access testingTest each supported model and distinguish available, forbidden, quota, timeout, and failed results.
Automatic monitoringRun scheduled checks globally and configure a separate interval for every provider.
Protected credentialsStore API keys encrypted and reveal them only when explicitly requested.

Recommended workflow

A practical sequence for adding and validating a provider.

  1. 1
    Add a provider

    Enter a provider name, base URL, API key label, optional API key, timeout, and monitoring interval.

  2. 2
    Test the connection

    Verify the endpoint and credential before saving the provider configuration.

  3. 3
    Load models

    Detect compatibility and retrieve the models currently exposed by the provider.

  4. 4
    Test model access

    Send minimal inference requests to determine which models accept the registered credential.

  5. 5
    Review available models

    Use the Available Models menu to compare successful model access across providers.

Security and privacy

Designed for private administration through a trusted host.

API keys are encrypted at rest using the configured master key.
Credentials are decrypted only for provider requests or an explicit Show API key action.
The recommended deployment binds to localhost and is accessed through an SSH tunnel.
Model access tests send real inference requests and may consume provider quota or credit.
